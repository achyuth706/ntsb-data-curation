# NTSB Aviation Accident Database — Data Curation Project

IS547 Foundations of Data Curation, Spring 2026

## About the Dataset

The NTSB (National Transportation Safety Board) publishes aviation accident data through their public data portal at https://data.ntsb.gov/avdata. The data covers civil aviation accidents investigated in the United States from 2008 onward and is updated periodically.

The data is distributed as zip archives. Each archive contains a single Microsoft Access database file (.mdb). Inside each .mdb file there are around 20 tables that together describe each accident — the event itself, the aircraft involved, injuries, crew information, engine details, and written narratives.

All tables are linked by a shared field called ev_id, which is the unique identifier for each accident event.

## What This Repo Contains

The goal of this project is to extract the data from the .mdb format into plain CSV files, apply basic cleaning, and document what the data looks like so it can be understood and reused by others.

The repo is organized as follows:

- `data/raw/` — original .mdb files (not committed due to size; download from NTSB portal)
- `data/processed/` — cleaned CSV exports of the main tables
- `data/reference/` — lookup and reference tables exported as-is
- `scripts/` — Python extraction and cleaning script
- `docs/` — data dictionary and other documentation
- `logs/` — provenance log and cleaning report generated by the script

## Tables

The main tables extracted are:

- events — one row per accident, contains date, location, weather, and injury totals
- aircraft — one row per aircraft involved, contains make/model, damage level, operator info
- injury — injury counts broken down by severity and person type
- narratives — written accident descriptions and probable cause statements
- flight_crew — pilot and crew information including certifications and experience
- engines — engine type and specifications

## How to Run

You need mdbtools installed (sudo apt install mdbtools on Linux/WSL) and pandas (pip install pandas).
```
python scripts/ntsb_extract.py --input data/raw/up01DEC.mdb --output data/
```

The script extracts all tables to CSV, applies basic cleaning, and writes a log to logs/.
