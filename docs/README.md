# docs/ — documentation index

**Project:** NTSB Aviation Accident Database Curation  
**Course:** IS547 Foundations of Data Curation, Spring 2026  
**Batch 2 completed:** 2026-05-05

This folder contains all documentation artifacts derived from the curated dataset.
The table below describes each file and its purpose.

---

## Completed (Batch 2)

| File | Description |
|---|---|
| [data_dictionary.md](data_dictionary.md) | Auto-generated field-level metadata for all six core tables. For each column: inferred data type, description from the NTSB *eADMSPUB_DataDictionary*, missing-value percentage, and notes on keys and coded fields. Generated by `scripts/generate_docs.py`. |
| [controlled_vocabularies.md](controlled_vocabularies.md) | Auto-generated catalogue of every coded field across the six core tables, with codes and meanings drawn verbatim from the NTSB data dictionary. Supports downstream mapping and interoperability (Wilkinson et al., 2016). Generated by `scripts/generate_docs.py`. |
| [data_quality_notes.md](data_quality_notes.md) | Interpretive companion to `logs/data_quality.md`. Explains why fields are null (investigation status vs. schema artefact), documents three key curation decisions (preserve missingness, no recoding, keep-latest dedup), and lists known limitations for reuse. |
| [narrative_field_notes.md](narrative_field_notes.md) | Field-level assessment of the four free-text narrative columns (`narr_accp`, `narr_accf`, `narr_cause`, `narr_inc`). Covers population rates, length variability, why `narr_inc` is structurally all-null, interpretive limitations, and reuse guidance. |
